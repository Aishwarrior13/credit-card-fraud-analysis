# Credit Card Fraud Analysis 🔍

## Overview
Analysis of 284,000+ credit card transactions to identify 
fraud patterns using Power BI.

## Problem Statement
Only 0.17% of transactions are fraudulent — detecting them 
is a classic needle-in-a-haystack problem requiring pattern 
analysis across time and amount dimensions.

## Key Insights
- $500–$1000 transactions have the highest fraud rate (0.4%)
- Early morning hours show highest fraud rate (0.45%) 
  despite lowest transaction volume
- Night has highest fraud volume due to overall higher 
  transaction activity
- Fraudsters prefer small amounts in volume but target 
  mid-range for highest rate

## Tools Used
- Power BI Desktop
- DAX measures
- Time bucket analysis
- Amount group binning
